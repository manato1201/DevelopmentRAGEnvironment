1mcp-rag-server のフォーク方針

⚠
現状の問題: 現在は他者（karaage 氏）の個人リポジトリを clone して使っています。 元リポジトリが削除・非公開化された場合、または破壊的変更が入った場合に即座に影響を受けます。

リスク一覧

リスク内容深刻度
リポジトリ削除元オーナーが予告なく非公開・削除する可能性高
破壊的変更upstream の更新で動作が壊れる可能性中
商用監査自社管理外コードへの直接依存は証跡が取りにくい中
カスタマイズ独自改修を加えたい場合に管理できない中

フォーク vs 完全独立 — 方針選択

フォーク（現時点の推奨）

GitHub で Fork ボタンを押すだけ。upstream の更新を選択的に取り込める。MIT ライセンスで改造・商用利用自由。OSS の恩恵を維持しながらリスク回避できる。

完全独立（独自実装）

ブランド・コードを完全に自社管理したい場合。大幅な改修が必要になり、upstream の更新は自力でマージする必要がある。将来的に検討。

フォーク手順

  • GitHub でフォーク
    元リポジトリ（github.com/karaage0703/mcp-rag-server）を開き、右上の Fork ボタンをクリック。
    フォーク先: manato1201/mcp-rag-server（または任意のリポジトリ名）
  • ローカルの remote を変更 
    cd C:\Users\matuu\Desktop\GameDevelopment\mcp-rag-server

# 現在の remote を確認
git remote -v

# origin を自分のフォークに変更
git remote set-url origin https://github.com/manato1201/mcp-rag-server.git

# 元リポジトリを upstream として保持（更新取り込み用）
git remote add upstream https://github.com/karaage0703/mcp-rag-server.git

# 確認
git remote -v
  • フォーク済みリポジトリに push 
    git push origin main
  • （必要なとき）upstream の更新を取り込む 
    git fetch upstream
git merge upstream/main
# コンフリクトがあれば解消してから push
git push origin main

フォーク後の推奨対応

  • pyproject.toml の authors を自分の名前に更新（管理責任の明示）
  • psycopg2-binary を依存から削除（未使用。PostgreSQL は使っていない）
  • CHANGELOG.md を追加して独自変更を記録（監査・ロールバック対応）
  • GitHub リポジトリを Private に設定（商用ソースを非公開にしたい場合）
  • GitHub Actions で uv sync && pytest を設定（upstream マージ後の自動テスト）
ℹ
ライセンスについて: mcp-rag-server は MIT ライセンスです。フォーク・改造・商用利用は自由で、ソース開示義務もありません。唯一の義務は「配布時に LICENSE ファイルを同梱すること」だけです。

クラウドRAG セットアップ順序

  • Google AI Studio で Gemini API キーを取得
  • Notion で Integration Token を取得（既存のものを流用可）
  • Google Sheets を新規作成（空のまま）→ URLから SHEETS_ID をコピー
  • script.google.com で GAS プロジェクトを作成し gas_cloud_rag.js を貼り付け
  • スクリプトプロパティに各キー・ID をすべて設定
  • GAS で testEmbedding() を実行 → ログに「次元数: 768」が出れば OK
  • GAS で syncNotionToSheets() を実行（初回は権限承認、2〜5分）
  • GAS で「デプロイ」→「新しいデプロイ」→ ウェブアプリ → URL を取得
  • Unity / Houdini の Settings タブに WebApp URL を入力

環境変数のセット方法

# セッション中のみ有効（PowerShell）
$env:ANTHROPIC_API_KEY = "sk-ant-..."

# システム環境変数に永続登録（管理者権限不要）
[System.Environment]::SetEnvironmentVariable("ANTHROPIC_API_KEY", "sk-ant-...", "User")
⚠
.env ファイルは使用しない。
全スクリプトから load_dotenv() は削除済みです。環境変数に直接セットしてください。.env を使うとキーがテキストファイルに保存され、git に誤コミットするリスクがあります。
