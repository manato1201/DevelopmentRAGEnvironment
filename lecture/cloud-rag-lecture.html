なぜGemini UIから直接呼べないのか

gemini.google.com は外部のGASエンドポイントをツールとして登録する機能を持たない。GASでWebAppを立て、そこからGemini APIを呼ぶ構成（A方式）が実装量最小かつ保守コストゼロ。

summaryが重要な理由

ページ本文をそのままGeminiに渡すとトークン数が増えコストが跳ね上がる。100字のsummaryを渡すことでコストを1/10以下に抑えながら精度を維持できる。追加時点でGeminiに生成させておくのがポイント。

database_idの確認方法

各DBのURLから取得する（GASのスクリプトプロパティに設定する）：

https://www.notion.so/{workspace}/xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx?v=...
                                   ← ここが database_id（32文字）

Step 04

Notion検索関数の実装

GASエディタの Code.gs に実装する。database_idを指定して検索することで、DB間の混合を防ぐ。

Code.gs — Notion検索
// DB識別キー → スクリプトプロパティキーのマップ
const DB_KEYS = {
  tool_docs:  'DB_TOOL_DOCS',
  game_info:  'DB_GAME_INFO',
  research:   'DB_RESEARCH',
  team_notes: 'DB_TEAM_NOTES'
};

function searchNotion(query, dbKey) {
  const props  = PropertiesService.getScriptProperties();
  const notionKey = props.getProperty('NOTION_API_KEY');
  const dbId   = props.getProperty(DB_KEYS[dbKey]);

  const payload = {
    filter: {
      or: [
        { property: 'title',   title:        { contains: query } },
        { property: 'summary', rich_text:    { contains: query } },
        { property: 'tags',    multi_select: { contains: query } }
      ]
    },
    page_size: 5  // 上位5件
  };

  const res = UrlFetchApp.fetch(
    `https://api.notion.com/v1/databases/${dbId}/query`,
    {
      method: 'post',
      headers: {
        'Authorization':  `Bearer ${notionKey}`,
        'Notion-Version': '2022-06-28',
        'Content-Type':   'application/json'
      },
      payload: JSON.stringify(payload)
    }
  );

  const data = JSON.parse(res.getContentText());
  if (!data.results?.length) return '関連ドキュメントが見つかりませんでした。';

  // summary を連結して context を構築
  return data.results.map((page, i) => {
    const title   = page.properties.title?.title?.[0]?.plain_text ?? '無題';
    const summary = page.properties.summary?.rich_text?.[0]?.plain_text ?? '（要約なし）';
    const url     = page.properties.source_url?.url ?? '';
    return `[${i+1}] ${title}\n${summary}${url ? `\n参照: ${url}` : ''}`;
  }).join('\n\n');
}
Step 05

Gemini呼び出し・RAGメイン処理

Code.gs — Gemini + RAG
function callGemini(prompt) {
  const apiKey = PropertiesService.getScriptProperties().getProperty('GEMINI_API_KEY');
  const url = `https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent?key=${apiKey}`;

  const res = UrlFetchApp.fetch(url, {
    method: 'post',
    contentType: 'application/json',
    payload: JSON.stringify({
      contents: [{ parts: [{ text: prompt }] }],
      generationConfig: { temperature: 0.3 }  // RAG用途は低めに
    })
  });

  return JSON.parse(res.getContentText())
    .candidates?.[0]?.content?.parts?.[0]?.text ?? '回答を取得できませんでした。';
}

function ragQuery(query, dbKey) {
  const context = searchNotion(query, dbKey);

  const prompt = `以下のドキュメントを参考に、日本語で正確に回答してください。
ドキュメントに情報がない場合はその旨を伝えてください。

【参考ドキュメント】
${context}

【質問】
${query}`;

  return callGemini(prompt);
}

// WebApp エントリポイント
function doPost(e) {
  try {
    const { query, dbKey } = JSON.parse(e.postData.contents);
    return ContentService
      .createTextOutput(JSON.stringify({ answer: ragQuery(query, dbKey), status: 'ok' }))
      .setMimeType(ContentService.MimeType.JSON);
  } catch (err) {
    return ContentService
      .createTextOutput(JSON.stringify({ answer: `エラー: ${err.message}`, status: 'error' }))
      .setMimeType(ContentService.MimeType.JSON);
  }
}

Troubleshooting

トラブルシューティング

Notion APIが403を返す

インテグレーションが各DBの「接続先」に追加されているか確認する。Notion DB → 右上「…」→「接続先」→ インテグレーションを追加。

GASでエラーが出る

GAS エディタ — デバッグ
// 関数を選択して「実行」→「実行ログ」でエラー確認
function debugSearch() {
  Logger.log(searchNotion('Unity', 'tool_docs'));
}

Geminiの回答が的外れ

summaryプロパティが空のページが多い場合に起きやすい。各ページにsummaryを追記すると精度が上がる。またクエリをより具体的にすること（「Unity」より「Unity HDRP シェーダーグラフ」）。

WebAppにアクセスできない

デプロイ時のアクセス権設定を確認。スクリプトプロパティに全キーが設定されているか確認。GASエディタで doGet() を直接実行してプレビューを確認する。
