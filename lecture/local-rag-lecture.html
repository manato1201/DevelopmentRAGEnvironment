ローカルRAG環境構築
WSL2 + Docker + mcp-rag-server

Windows PCに自前のベクトル検索エンジンを立て、Claude CodeをはじめとするMCPホストから プライベートドキュメントを参照できるようにする手順書。

Prerequisites

前提条件

必要な環境
  • Windows 11
  • Claude Desktop インストール済み
  • Houdini インストール済み（helpデータ取得用）
  • ディスク空き 10GB以上
不要なもの
  • Docker Desktop（WSL内に直接インストール）
  • Python の別途インストール（uv が管理）
  • PostgreSQL のWindowsネイティブ版

なぜWSL2内にDockerを入れるのか

Docker DesktopはGUI管理ツールとして便利だが、WSL2内でDockerエンジンを直接動かす方がリソース消費が少なく、mcp-rag-serverとの連携がシンプルになる。

Step 01

WSL2セットアップ

1
Ubuntuをインストール PowerShell（管理者）
PowerShell — 管理者
PS> wsl --install -d Ubuntu

インストール完了後、Ubuntuが起動するのでユーザー名とパスワードを設定する。再起動を求められた場合は再起動してから再度Ubuntuを起動する。

2
systemdの有効化 Ubuntu
Ubuntu
$ sudo tee /etc/wsl.conf << 'EOF'
[boot]
systemd=true
EOF

設定反映のためWSLを再起動する：

PowerShell
PS> wsl --shutdown
3
パッケージ更新 Ubuntu
Ubuntu
$ sudo apt update && sudo apt upgrade -y
Step 02

Dockerインストール

1
必要パッケージ Ubuntu
Ubuntu
$ sudo apt install -y ca-certificates curl gnupg lsb-release
2
Docker公式GPGキーを追加 Ubuntu
Ubuntu
$ sudo install -m 0755 -d /etc/apt/keyrings
$ curl -fsSL https://download.docker.com/linux/ubuntu/gpg \
    | sudo tee /etc/apt/keyrings/docker.asc > /dev/null
$ sudo chmod a+r /etc/apt/keyrings/docker.asc
3
Dockerリポジトリを追加 Ubuntu
Ubuntu
$ echo \
  "deb [arch=$(dpkg --print-architecture) \
  signed-by=/etc/apt/keyrings/docker.asc] \
  https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo "$VERSION_CODENAME") stable" \
  | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
4
Dockerをインストール Ubuntu
Ubuntu
$ sudo apt update
$ sudo apt install -y docker-ce docker-ce-cli containerd.io \
    docker-buildx-plugin docker-compose-plugin
5
sudoなしで使えるようにする Ubuntu
Ubuntu
$ sudo usermod -aG docker $USER
$ newgrp docker
6
動作確認 Ubuntu
Ubuntu
$ docker --version
Docker version 27.x.x ...
$ docker run hello-world
Hello from Docker!  ← これが出れば成功
Step 03

pgvector起動

PostgreSQL + pgvectorをDockerコンテナで起動する。ベクトル検索のデータストアになる。

1
コンテナ起動 Ubuntu
Ubuntu
$ docker run --name postgres-pgvector \
  -e POSTGRES_PASSWORD=password \
  -p 5432:5432 \
  -d pgvector/pgvector:pg17
2
データベース作成 Ubuntu
Ubuntu
$ docker exec -it postgres-pgvector \
  psql -U postgres -c "CREATE DATABASE ragdb;"
3
確認 Ubuntu
Ubuntu
$ docker ps
CONTAINER ID   IMAGE                    STATUS
a1b2c3d4e5f6   pgvector/pgvector:pg17   Up 2 minutes
↑ Up と表示されていれば成功

Tips

WSL再起動後にコンテナが停止している場合は docker start postgres-pgvector で再起動できる。

Step 04

mcp-rag-serverセットアップ

1
uvのインストール Ubuntu
Ubuntu
$ curl -LsSf https://astral.sh/uv/install.sh | sh
$ source $HOME/.local/bin/env

uv はPythonの高速パッケージマネージャー。pipより依存解決が安定している。

2
リポジトリをクローン Ubuntu
Ubuntu
$ cd ~
$ git clone https://github.com/karaage0703/mcp-rag-server
$ cd mcp-rag-server
$ uv sync
3
.envファイルを作成 Ubuntu
Ubuntu — ~/mcp-rag-server
$ tee .env << 'EOF'
POSTGRES_HOST=localhost
POSTGRES_PORT=5432
POSTGRES_USER=postgres
POSTGRES_PASSWORD=password
POSTGRES_DB=ragdb

SOURCE_DIR=./data/source
PROCESSED_DIR=./data/processed

EMBEDDING_MODEL=intfloat/multilingual-e5-large
EMBEDDING_DIM=1024
EMBEDDING_PREFIX_QUERY="query: "
EMBEDDING_PREFIX_EMBEDDING="passage: "
EOF
4
確認 Ubuntu
Ubuntu
$ cat .env
POSTGRES_HOST=localhost
...
Step 05

Houdiniヘルプデータの配置

この工程について

HoudiniのヘルプデータをローカルRAGのソースとして使う。zipをWSL側にコピーして展開し、.txtファイルのみ残す。

1
ソースディレクトリ作成 Ubuntu
Ubuntu
$ mkdir -p ~/mcp-rag-server/data/source
2
Houdini helpのzipを探す Windows

エクスプローラーで以下を開く（バージョン番号は環境に合わせて変更）：

C:\Program Files\Side Effects Software\Houdini 21.0.506\houdini\help
3
WSLのsourceフォルダへコピー Windows

エクスプローラーのアドレスバーに以下を貼り付けてアクセス：

\\wsl.localhost\Ubuntu\home\tk_render\mcp-rag-server\data\source

コピーするzipファイル：

ファイル名内容優先度
nodes.zipノードリファレンス★★★ 最重要
expressions.zip式・関数★★★
commands.zipHScriptコマンド★★★
hapi.zipHoudini Engine API★★☆
vex.zipVEXリファレンス★★☆
basics.zip基本操作★☆☆ 余裕があれば
render.zipレンダリング★☆☆ 余裕があれば
network.zipネットワーク操作★☆☆ 余裕があれば
4
zip展開 Ubuntu

scripts/extract_zip.py を実行する：

Ubuntu — ~/mcp-rag-server
$ python3 scripts/extract_zip.py
  ✓ 展開完了: nodes.zip
  ✓ 展開完了: expressions.zip
  ...
5
.txt以外を削除 Ubuntu
Ubuntu — ~/mcp-rag-server
$ python3 scripts/delete_non_txt.py
.txt以外のファイルを削除します...
続行しますか？ (y/N): y
完了: 1842件削除 / 0件エラー
6
確認 Ubuntu
Ubuntu
$ find data/source -name "*.txt" | wc -l
4823   ← 数百〜数千あれば OK
Step 06

インデックス化

所要時間に注意

初回インデックス化は4時間以上かかる。就寝前や外出前など、PCを長時間放置できるタイミングで実行すること。差分インデックス機能があるため、途中で止まっても再実行すれば続きから処理される。

Ubuntu — ~/mcp-rag-server
# バックグラウンド実行
$ nohup uv run python -m src.cli index > index.log 2>&1 &
PID: 12345

# 進捗確認
$ tail -f index.log

# 処理済みファイル数の確認
$ grep "INFO" ~/mcp-rag-server/index.log | wc -l
Step 07

MCPサーバー設定

1
MCPサーバーの動作確認 Ubuntu
Ubuntu — ~/mcp-rag-server
$ uv run python -m src.main
Starting MCP server...  ← この表示が出れば OK、Ctrl+C で止める
2
Claude Desktopの設定ファイルを開く Windows

Everythingで以下を検索して開く：

claude_desktop_config.json
3
設定ファイルを編集 Windows

以下に書き換える。tk_render を自分のUbuntuユーザー名に変更すること：

claude_desktop_config.json
{
  "mcpServers": {
    "mcp-rag-server": {
      "command": "wsl",
      "args": [
        "bash",
        "-c",
        "/home/tk_render/.local/bin/uv run --directory /home/tk_render/mcp-rag-server python -m src.main"
      ]
    }
  }
}
4
動作確認 Claude Desktop

Claude Desktopを完全再起動後、以下で確認：

確認プロンプト

mcp-rag-serverで "vellum solver" について検索して、パラメータ一覧と使い方を英語で検索し日本語で教えてください。

Claude Desktopの設定で running になっていれば完了。

Troubleshooting

トラブルシューティング

Dockerコンテナが起動しない

Ubuntu
$ docker ps -a          # 状態確認
$ docker start postgres-pgvector  # 再起動
$ docker logs postgres-pgvector   # ログ確認

uvコマンドが見つからない

Ubuntu
$ source $HOME/.local/bin/env

# .bashrc に追記して永続化
$ echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
$ source ~/.bashrc

Claude DesktopでMCPがrunningにならない

  • claude_desktop_config.json のユーザー名が正しいか確認
  • Ubuntuで uv run python -m src.main が単体で動くか確認
  • pgvectorコンテナが起動中か docker ps で確認
  • Claude Desktopをタスクトレイから完全終了して再起動

インデックス化が止まった

Ubuntu
$ ps aux | grep "src.cli"   # プロセス確認

# 再実行（差分なので途中から再開される）
$ cd ~/mcp-rag-server
$ nohup uv run python -m src.cli index > index.log 2>&1 &
