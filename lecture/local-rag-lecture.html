Setup Guide · Local RAG · ChromaDB版

ローカルRAG環境構築
ChromaDB + Windows Native + mcp-rag-server

Docker・WSL2不要。Windows PCにChromaDBベースのベクトル検索エンジンを立て、 Obsidian vaultと連携してプライベートドキュメントをClaude Desktopから参照できるようにする手順書。

Prerequisites

前提条件

必要な環境
  • Windows 11
  • Claude Desktop インストール済み
  • Obsidian インストール済み
  • ディスク空き 5GB 以上
不要なもの（旧構成から削除）
  • WSL2 / Ubuntu
  • Docker Desktop
  • PostgreSQL / pgvector
  • Python の別途インストール（uv が管理）

なぜ ChromaDB に移行したのか

pgvector は Docker + WSL2 が必須でセットアップが複雑。ChromaDB はファイルベースのローカルDB（SQLite + HNSW）で動くため、Dockerなしで同等の検索精度が得られる。個人・小規模チームのプライベートRAGには最適。

比較項目旧構成（pgvector）新構成（ChromaDB）
Docker必要不要
WSL2必要不要
セットアップ時間1時間以上約20分
検索精度cosine similaritycosine similarity（同等）
データ保存PostgreSQL（コンテナ内）ローカルフォルダ（./data/chroma）
Namespaceテーブル分離Collection分離（Obsidianフォルダ対応）
Step 01

uv インストール

uv は Python の高速パッケージマネージャー。pip より依存解決が安定しており、Python バージョン管理も一括して行える。

1
winget でインストール PowerShell
PowerShell
PS> winget install --id=astral-sh.uv -e
2
バージョン確認 PowerShell
PowerShell
PS> uv --version
uv 0.x.x  ← バージョンが表示されれば OK
Step 02

mcp-rag-server セットアップ

1
リポジトリをクローン PowerShell
PowerShell
PS> cd C:\Users\YOUR_USERNAME\Desktop\GameDevelopment
PS> git clone https://github.com/karaage0703/mcp-rag-server
PS> cd mcp-rag-server
2
依存関係インストール（Python 3.12 指定） PowerShell
PowerShell — mcp-rag-server
PS> uv sync --python 3.12
PS> uv add chromadb watchdog pyyaml

Python 3.12 を必ず指定すること

uv sync のみだと Python 3.13 が選択され、sentencepiece のビルドが CMake エラーで失敗する。3.12 には pre-built wheel があるためビルド不要。

Step 03

ChromaDB パッチの適用

mcp-rag-server の vector_database.py と rag_tools.py を ChromaDB 対応版に差し替える。

1
vector_database.py を差し替え PowerShell
PowerShell — mcp-rag-server
PS> copy ..\DevelopmentRAGEnvironment\scripts\vector_database.py src\vector_database.py

差し替えた vector_database.py の主な変更点：

  • PostgreSQL 接続コードを全削除 → chromadb.PersistentClient に置き換え
  • Obsidian の最上位フォルダ名 → ChromaDB Collection 名（namespace）にマッピング
  • _rag_dashboard、_templates、.processed フォルダを自動スキップ
  • cosine similarity（距離 → 類似度変換）対応
2
rag_tools.py を編集 テキストエディタ

src\rag_tools.py を開き、create_rag_service_from_env 関数内を以下のように変更：

変更前
vector_database = VectorDatabase({
    "host":     os.environ.get("POSTGRES_HOST",     "localhost"),
    "port":     os.environ.get("POSTGRES_PORT",     "5432"),
    "user":     os.environ.get("POSTGRES_USER",     "postgres"),
    "password": os.environ.get("POSTGRES_PASSWORD", "password"),
    "database": os.environ.get("POSTGRES_DB",       "ragdb"),
})
変更後
vector_database = VectorDatabase({
    "chroma_path":   os.environ.get("CHROMA_PATH",   "./data/chroma"),
    "embedding_dim": os.environ.get("EMBEDDING_DIM", "1024"),
})
3
.env ファイルを作成 PowerShell
PowerShell — mcp-rag-server
PS> copy ..\DevelopmentRAGEnvironment\.env.example .env

.env の内容（YOUR_USERNAME を自分のユーザー名に変更）：

# ChromaDB データ保存先
CHROMA_PATH=./data/chroma

# Obsidian vault のパス
SOURCE_DIR=C:\Users\YOUR_USERNAME\Desktop\GameDevelopment\DevelopmentRAGEnvironment\localRAG
PROCESSED_DIR=C:\Users\YOUR_USERNAME\Desktop\GameDevelopment\DevelopmentRAGEnvironment\localRAG\_rag_dashboard\.processed

# 埋め込みモデル（変更不要）
EMBEDDING_MODEL=intfloat/multilingual-e5-large
EMBEDDING_DIM=1024
EMBEDDING_PREFIX_QUERY=query:
EMBEDDING_PREFIX_EMBEDDING=passage:
Step 04

Obsidian vault の設定

Obsidian フォルダ構成 = ChromaDB namespace

vault 内の最上位フォルダ名がそのまま ChromaDB の Collection 名（namespace）になる。_ や . で始まるフォルダは管理用として自動スキップされる。

1
vault フォルダ構成
DevelopmentRAGEnvironment\localRAG\   ← Obsidian で開くフォルダ
├── chat_logs\        ← namespace: chat_logs
├── tutorials\        ← namespace: tutorials
├── personal_notes\   ← namespace: personal_notes
│   ├── progress\
│   └── ideas\
├── private_docs\     ← namespace: private_docs
├── _rag_dashboard\   ← 管理専用（インデックス対象外）
│   ├── index_status.md
│   ├── namespace_map.md
│   └── .processed\
└── _templates\       ← テンプレート（インデックス対象外）
2
Obsidian でフォルダを開く Obsidian

Obsidian を起動 → 「別のボルトを開く」→ フォルダを選択 → localRAG\ を指定。

3
テンプレートプラグインを有効化 Obsidian

設定 → コアプラグイン → テンプレート: オン

設定 → テンプレート → テンプレートフォルダの場所: _templates

YAML フロントマターに以下を含めるとステータス管理が可能：

---
status: active   # active / stale / archived
rag_indexed: false
expires: 2026-12-31  # オプション
---
Step 05

インデックス化

旧構成との違い

pgvector 版では 4 時間以上かかる初回インデックス化が必要だったが、ChromaDB 版は Obsidian vault が空のままインデックス化を実行でき（0件で正常終了）、ノートを追加するたびに差分インデックスすれば良い。

PowerShell — mcp-rag-server
PS> uv run python -m src.cli index
インデックス化が完了しました
- 処理ファイル数: 1
- 総チャンク数: 8   ← ファイルを追加後の例

# 差分のみ更新する場合
PS> uv run python -m src.cli index --incremental

自動スキップされるフォルダ

_rag_dashboard\・_templates\・.processed\ は _namespace_from_path() で自動除外される。手動での除外設定は不要。

Step 06

watchdog（自動インデックス化）の起動

Obsidian でノートを保存するたびに、auto_index.py が変更を検知して自動でインデックスを更新する。

1
スクリプトをコピー PowerShell
PowerShell — mcp-rag-server
PS> copy ..\DevelopmentRAGEnvironment\scripts\auto_index.py .
2
バックグラウンドで起動 PowerShell
PowerShell — mcp-rag-server
PS> Start-Process -NoNewWindow -FilePath "uv" -ArgumentList "run python auto_index.py"

python auto_index.py では起動しないこと

uv の仮想環境外で実行すると ModuleNotFoundError: No module named 'yaml' が発生する。必ず uv run 経由で起動すること。

3
動作確認 Obsidian

Obsidian でノートを保存後、_rag_dashboard\index_status.md のタイムスタンプが更新されれば watchdog が正常に動作している。

Step 07

Claude Desktop への登録

1
設定ファイルを開く Windows

Everything などで以下を検索して開く：

claude_desktop_config.json

通常のパス：

C:\Users\YOUR_USERNAME\AppData\Roaming\Claude\claude_desktop_config.json
2
設定を書き換える テキストエディタ

YOUR_USERNAME を自分のユーザー名に変更する：

claude_desktop_config.json
{
  "mcpServers": {
    "mcp-rag-server": {
      "command": "uv",
      "args": [
        "run",
        "--directory",
        "C:\\Users\\YOUR_USERNAME\\Desktop\\GameDevelopment\\mcp-rag-server",
        "python",
        "-m",
        "src.main"
      ]
    }
  }
}

旧構成との違い

旧版は "command": "wsl" で WSL を経由していたが、新版は "command": "uv" で Windows ネイティブに直接起動できる。

3
動作確認 Claude Desktop

Claude Desktop を完全再起動後、設定画面で running と表示されれば完了。

確認プロンプト

mcp-rag-server でインデックス内のドキュメント数を確認してください。

Troubleshooting

トラブルシューティング

sentencepiece のビルドが失敗する（CMake エラー）

Python 3.13 には sentencepiece の pre-built wheel がないためビルドが失敗する。

PowerShell
PS> uv sync --python 3.12   # Python バージョンを明示指定

'NoneType' object has no attribute 'get_or_create_collection'

vector_database.py の差し替えが正しく行われていない。

  • src\vector_database.py が ChromaDB 版に差し替えられているか確認
  • initialize_database() が self.connect() を呼んでいるか確認
  • 再度コピー: copy ..\DevelopmentRAGEnvironment\scripts\vector_database.py src\vector_database.py

ChromaDB コレクション名エラー（Got: C:\）

.env の SOURCE_DIR が正しく設定されていない。

  • .env の SOURCE_DIR が絶対パスで設定されているか確認
  • 設定後は .processed\ キャッシュを削除して再インデックス
PowerShell
PS> Remove-Item -Recurse -Force localRAG\_rag_dashboard\.processed
PS> uv run python -m src.cli index

watchdog が起動後すぐ終了する / yaml モジュールエラー

PowerShell
# NG — uv 仮想環境外で実行するとモジュールが見つからない
PS> python auto_index.py

# OK — uv run 経由で起動する
PS> Start-Process -NoNewWindow -FilePath "uv" -ArgumentList "run python auto_index.py"

Claude Desktop で MCP が running にならない

  • claude_desktop_config.json のパスが正しいか確認（\\ でエスケープ）
  • PowerShell で uv run --directory "C:\...\mcp-rag-server" python -m src.main が単体で動くか確認
  • Claude Desktop をタスクトレイから完全終了して再起動

インデックス化で _rag_dashboard が処理される

  • vector_database.py のパッチが正しく適用されているか確認
  • _namespace_from_path() が SOURCE_DIR 環境変数を読んで相対パスに変換しているはず
  • .env の SOURCE_DIR が設定されているか確認
